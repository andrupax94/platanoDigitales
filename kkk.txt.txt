Cuestionario CMS

    1. ¿Qué quiere decir CMS, en inglés y español?
Content Management System, Sistema de gestión de contenidos

    2. Qué es un CMS
Un CMS (Content Management System, por sus siglas en inglés) es un software utilizado para crear y gestionar el contenido de un sitio web de manera sencilla y eficiente. Proporciona una interfaz que permite a los usuarios sin conocimientos de programación realizar tareas como crear, editar, organizar y publicar contenido en un sitio web. 

    3. Para qué sirve un CMS
Un CMS (Content Management System, por sus siglas en inglés) sirve para facilitar la creación, gestión y publicación de contenido en un sitio web.

    4. Nombre 5 ventajas de los CMS
        1. Fácil gestión de los contenidos
        2. Son fácilmente personalizables y flexibles
        3. Seguridad de los datos.
        4. Optimización del sitio web para que sean accesibles a los motores de búsqueda, facilitando el CEO
        5. Proporcionan un entorno colaborativo para equipos de trabajos, además de ser facilmente escalables.
           
    5. 5 categorías de CMS (cuales son y para que se usan) WordPress. 
        1. De propósito general: Wordpress. Jomla, Drupal. Ideales para blogs, sitios web, portales de noticias.
        2. Comercio electrónico: Magento, Shopify y Woocomerce. Creación y gestión de tiendas on-line.
        3. Publicaciones y medios: Wordpress, Drupal y Ghost. Para la gestión de contenido editorial, publicación de noticias y artículos.
        4. E-learning: Modle, Canvas y Chamilo. Para la creación y gestión de plataformas educativas.
        5. Empresariales: Share Point, Sitecore y Liferay. Para grandes empresas que necesitan gestionar contenidos complejos y escalables y manejo de múltiples sitios web con una alta seguridad.

    8. ¿Es posible importar los contenidos actuales del antiguo sitio web a un CMS?
Sí, es posible importar los contenidos actuales de un antiguo sitio web a un CMS, aunque la facilidad y el proceso específico pueden variar según el CMS que estés utilizando y el formato de los contenidos existentes. 

    9. ¿Qué conocimientos son necesarios para administrar un CMS?
        ◦ Conocimientos básicos de informática
        ◦ Conocimeientos de gestión de contenidos
            ▪ Organización de páginas
            ▪ Gestión de menús
            ▪ Creación de categorías y etiquetas
            ▪ Gestión de medios: Imágenes, vídeos, documentos (pdf’s..)
        ◦ 	Conocimientos básicos de diseño web: html y css
        ◦ Conocimientos de seguridad
            ▪ Elección de contraseñas seguras
            ▪ Actualizaciones de seguridad
            ▪ Configuración de permisos de usuarios
            ▪ Medidas de protección contra malware y ataques
        ◦ Solución de problemas y mantenimiento del CMS
            ▪ Solución de errores
            ▪ Copias de seguridad
            ▪ Restauración de datos
            ▪ Actualizaciones del CMS y plugins
              
    10. ¿Qué se necesita para instalar un CMS?
        ◦ Servidor web o hosting
        ◦ Nombre de dominio
        ◦ Base de datos: Instalación de un servidor de base de datos y creación de la base de datos que va a usar el CMS
        ◦ Software del CMS
        ◦ Cliente FTP o Panel de Control
        ◦ Configuración inicial, dependiendo de las características de cada CMS.
